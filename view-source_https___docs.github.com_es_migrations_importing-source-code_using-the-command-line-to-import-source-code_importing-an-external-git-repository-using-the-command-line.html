data-search="article-body" class="Box-sc-g0xbh4-0 kohLxz"><div id="article-contents"><div class="MarkdownContent_markdownBody__v5MYy markdown-body"><p>Antes de comenzar, asegúrate de saber lo siguiente:</p>
<ul>
<li>Tu nombre de usuario GitHub</li>
<li>La dirección URL de clonación del repositorio externo, como <code>https://external-host.com/user/repo.git</code> o <code>git://external-host.com/user/repo.git</code> (quizás con un elemento <code>user@</code> delante del nombre de dominio <code>external-host.com</code>)</li>
</ul>
<div class="ghd-alert ghd-alert-success ghd-spotlight-success">
<p>A los fines de demostración, usaremos lo siguiente:</p>
<ul>
<li>Una cuenta externa denominada <strong>extuser</strong></li>
<li>Un host de Git externo denominado <code>https://external-host.com</code></li>
<li>Una cuenta personal de GitHub denominada <strong>ghuser</strong></li>
<li>Un repositorio en GitHub.com llamado <strong>repo.git</strong></li>
</ul>
</div>
<ol>
<li>
<p><a href="/es/repositories/creating-and-managing-repositories/creating-a-new-repository" _originalHref="/repositories/creating-and-managing-repositories/creating-a-new-repository">Creación de un repositorio en GitHub</a>. Importarás tu repositorio de Git externo a este repositorio nuevo.</p>
</li>
<li>
<p>En la línea de comandos, haz un clon "en blanco" del repositorio externo usando la URL del clon externo. Esto crea una copia completa de los datos, pero sin un directorio de trabajo para editar archivos, y asegura una exportación limpia y nueva de todos los datos antiguos.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_">$ </span><span class="bash">git <span class="hljs-built_in">clone</span> --bare https://external-host.com/EXTUSER/REPO.git</span>
<span class="hljs-meta prompt_"># </span><span class="bash">Makes a bare <span class="hljs-built_in">clone</span> of the external repository <span class="hljs-keyword">in</span> a <span class="hljs-built_in">local</span> directory</span>
</code></pre>
</li>
<li>
<p>Sube el repositorio clonado de forma local a GitHub usando la opción "espejo", que asegura que todas las referencias, como ramas y etiquetas, se copien en el repositorio importado.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_">$ </span><span class="bash"><span class="hljs-built_in">cd</span> REPO.git</span>
<span class="hljs-meta prompt_">$ </span><span class="bash">git push --mirror https://github.com/USER/REPO.git</span>
<span class="hljs-meta prompt_"># </span><span class="bash">Pushes the mirror to the new repository on GitHub.com</span>
</code></pre>
</li>
<li>
<p>Elimina el repositorio local temporal.</p>
<pre><code class="hljs language-shell">cd ..
rm -rf REPO.git
</code></pre>
</li>
</ol>
<p>Si el repositorio que vas a importar contiene archivos de gran tamaño, puede producirse una advertencia o un error. Para más información sobre los archivos grandes y cómo administrarlos, consulte "<a href="/es/repositories/working-with-files/managing-large-files/about-large-files-on-github" _originalHref="/repositories/working-with-files/managing-large-files/about-large-files-on-github">Acerca de los archivos grandes en GitHub</a>."</p>
